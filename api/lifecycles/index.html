lifecycles

Handles lifecycle method connections (init, start, etc.) for modules. Supports adding custom lifecycle methods and disconnecting them cleanly.

Functions​

register​

</>
lifecycles.register(
name: string--

the name of the lifecycle method (e.g., "shutdown")

) → ()

Adds a new lifecycle method name that modules can define and be invoked automatically.

bindSignal​

</>
lifecycles.bindSignal(
lifecycleName: string,--

the lifecycle method to invoke (e.g., "shutdown")

signal: RBXScriptSignal--

the RBXScriptSignal to connect

) → ()

Binds a lifecycle method to a RBXScriptSignal. When the signal fires, the method will be invoked on all connected modules that define it.

connect​

</>
lifecycles.connect(
name: string,--

the name of the module (used as key)

module: types.lifecycleModule,--

the module to connect

waitForFn: ((name: string) → any)?--

optional waitFor function to inject

) → {[string]: any}--

promises for each lifecycle stage

Connects and runs all registered lifecycle methods on the given module. Returns a table of promises keyed by lifecycle name.

disconnect​

</>
lifecycles.disconnect(
name: string--

the module name to disconnect

Disconnects lifecycle promises and cleanup for a given module by name.
